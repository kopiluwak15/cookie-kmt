## 1. システム概要

縮毛矯正・髪質改善専門サロン「COOKIE 熊本」の業務管理システムです。

**主な目的**: 「悩み → 施術 → 再来店」の再現性を可視化し、スタッフの努力の方向性を見える化する。

**できること**:
- 顧客カルテ管理（来店履歴・施術記録・悩み・ケース記録）
- LINE連携による自動リマインド配信（サンキュー・リマインド・休眠・メンテナンス）
- スタッフの施術ログ記録・実績管理・報酬計算
- 出退勤管理（GPS位置確認付き）
- 売上分析・リピート率分析
- 店舗iPadでの顧客カルテ閲覧（QR + LINE認証 + GPS制限）

**対応デバイス**: PC（管理者）、タブレット（iPad）、スマートフォン（LINE内ブラウザ）

## 2. ログイン・認証

### ログイン方法

1. https://kmt.cookie.hair にアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」をクリック

- **管理者（admin）**: ログイン後にダッシュボードへ遷移
- **スタッフ（staff）**: ログイン後にマイ実績ページへ遷移

### 初回ログイン時

管理者が登録した仮パスワードでログインすると、パスワード変更画面が表示されます。新しいパスワードを設定してください。

### LINE連携（スタッフ）

マイページから「LINE連携」ボタンでLINEアカウントと紐づけできます。連携すると出退勤のLIFF（LINE内ブラウザ）機能が使えるようになります。

## 3. スタッフ機能

左のサイドバーから各機能にアクセスできます。

### 3-1. 施術ログ (`/staff/visit-log`)

施術後に入力する記録フォームです。

**入力項目**:
- **担当スタッフ**: ログイン中のスタッフ名が自動設定
- **スタイルカテゴリ**: ツーブロック・マッシュ等から選択
- **サービスメニュー**: カット・カラー・ヘッドスパ等から選択
- **価格**: 施術金額を入力
- **使用薬剤**: 4カテゴリ（ストレート・カラー・パーマ・トリートメント）から選択
  - 薬剤の選択肢は管理者が設定画面から追加・削除できます

**操作手順**:
1. 本日チェックイン済みの顧客を選択
2. 各項目を入力
3. 「保存」で記録完了

---

### 3-2. カウンセリング (`/staff/counseling`)

本日チェックイン済みの顧客を管理する画面です。

**2つのタブ**:
- **カルテ未確認**: チェックインしたがカルテ（初回アンケート）をまだ確認していない顧客。「アンケート再送」ボタンで顧客にLINEでアンケートを再送信できます。
- **カルテ確認済み**: カルテを閲覧済みの顧客一覧。

---

### 3-3. 店舗売上 (`/staff/store-sales`)

全スタッフの店舗合計売上を閲覧できます（閲覧のみ、編集不可）。

**表示内容**:
- 今月のKPI 4項目（売上・客数・客単価・平均施術時間）
- スタッフ別実績テーブル（当月）
- 月別実績テーブル（過去12ヶ月）

---

### 3-4. マイ実績 (`/staff/performance`)

自分の施術実績と報酬予測を確認できます。

**表示内容**:

| 項目 | 説明 |
|------|------|
| 売上（税込/税抜） | 当月の施術売上合計 |
| 客数 | 当月の担当客数 |
| 客単価 | 売上 ÷ 客数 |
| 平均施術時間 | 1施術あたりの平均所要時間 |
| 平均60分基準タイム | 施術時間を予定所要時間で正規化した効率指標 |

**報酬予測（雇用形態別）**:

| 雇用形態 | 表示内容 |
|----------|---------|
| 業務委託 | 税抜売上 × 歩合率 → 源泉徴収（10.21%）控除後の手取り予測 |
| 正社員 S1〜S3 | 基本給 + 次ステージ昇格条件のプログレスバー |
| 正社員 S4 | 基本給 + 月売上60万超過分×10%のインセンティブ（2ヶ月連続条件） |
| 正社員 S5・S6 | 基本給（S5は役職手当込み） |
| パート | 時給・施術数・売上 |

**月別実績テーブル**: 過去12ヶ月分。各月をクリックすると来店一覧（顧客名・メニュー・スタイル・価格）を展開表示。

---

### 3-5. マイページ (`/staff/mypage`)

個人プロフィールの確認ページです。

**表示項目**: 氏名・メール・雇用形態・ステージ・報酬情報（歩合率/基本給/時給）

**業務委託スタッフのみ追加表示**:
- 業務委託契約書（署名状況の確認）
- ルールブック同意書（同意状況の確認）
- 美容師免許（画像アップロード機能）

---

### 3-6. 出退勤（タイムカード）

LINEのリッチメニューから出退勤を記録します。

**出勤の流れ**:
1. LINEのリッチメニューから「出勤」をタップ
2. LIFF（LINE内ブラウザ）が開く
3. GPS位置確認（店舗の許容半径内かチェック）
4. 範囲内であれば出勤打刻完了

**退勤の流れ**: 出勤と同様。退勤時にチェックリスト（清掃・戸締まり等）の確認画面が表示される場合があります。

**GPS範囲外の場合**: 「店舗内でのみ使用できます」というエラーが表示され、打刻できません。

## 4. 管理者機能

管理者（admin）でログインすると、左サイドバーに追加の管理メニューが表示されます。

### 4-1. ダッシュボード (`/admin/dashboard`)

店舗の経営状況を一覧で確認できます。

**期間フィルター**: 直近30日・今月・先月・3ヶ月・1年、または日付手動指定

**KPI（上段4項目）**:
- 総顧客数
- LINE連携数
- 来店数
- 売上

**KPI（下段4項目）**:
- 新規顧客数
- 平均施術時間
- 休眠顧客数（90日以上未来店、橙色で警告）
- 失客予備軍（60〜90日未来店、赤色で警告）

**チャート・テーブル**:
- 来店動機チャート（円グラフ/棒グラフ）
- スタイル別来店数テーブル
- LINE配信状況テーブル（サンキュー・リマインド①②・休眠の送信数と成功率）

---

### 4-2. 顧客管理 (`/admin/customers`)

**顧客一覧**:
- 顧客ID・氏名・電話番号・来店回数・最終来店日・LINE連携状況
- リアルタイム検索（名前・ID・電話番号で絞り込み）

**LINE状況バッジ**:
| バッジ | 意味 |
|--------|------|
| 連携済（緑） | LINE連携済み、配信可能 |
| 未連携（灰） | LINE未連携 |
| ブロック（赤） | LINEブロック済み、配信不可 |

**顧客手動追加**: 「＋顧客追加」ボタンから追加。名前（必須）・電話番号・来店経路を入力。LINE未登録の顧客を本日チェックイン済みとして登録できます。

**顧客詳細ページ** (`/admin/customers/[id]`):

顧客IDをクリックすると詳細ページに遷移。以下のタブで構成:

| タブ | 内容 |
|------|------|
| 来店履歴 | 日付・担当・メニュー・スタイル・価格の一覧 |
| カルテ | 初回アンケート回答内容の閲覧 |
| ケース記録 | 施術ごとの悩み・施術内容・薬剤・AI要約 |
| LINE送信履歴 | サンキュー・リマインド等の送信ログ |

### 4-5. 労務管理

3つのサブページがあります。

#### お知らせ送信 (`/admin/labor-management/announcements`)

LINEでスタッフに一斉通知を送信します。

**設定項目**:
- 重要度: 重要🔴・確認🟠・指示🟡・お知らせ🟢・その他
- タイトル・本文
- 配信タイミング: 「チェックイン時に表示」または「即時送信」
- 受信スタッフ: チェックボックスで選択

**テンプレート機能**: よく使う通知内容をテンプレートとして保存・呼び出しできます。

**送信済み一覧**: 送信済みお知らせの一覧と、未確認スタッフの名前を表示。

#### タイムカード (`/admin/labor-management/timecard`)

**スタッフ別・月別の打刻記録を確認**:
- 出勤時刻・退勤時刻・GPS検証状況（verified/未検証）
- 管理者による打刻時刻の手動編集・削除（パスワード確認あり）
- CSV出力（施術ログ含む/含まないのオプション）
- GPS機能のON/OFF切替スイッチ

#### チェックリスト (`/admin/labor-management/checklist`)

開店・閉店チェックリストの管理画面です。

## 5. 顧客向け機能（LINE経由）

顧客はLINEからQRコードを読み取ることでシステムを利用します。

### 5-1. 初回来店（QR → カルテ登録）

1. 店舗のQRコードをLINEで読み取る
2. LINE認証が行われる
3. 公式LINE友だち追加の確認画面（未追加の場合）
4. 初回の場合、カルテ登録フォームに遷移

**カルテ登録フォームの入力項目**:
- 来店経路（ホットペッパー・紹介・Instagram・看板・Google・その他）
- 職業
- 住所（熊本市各区〜熊本県外の選択）
- 髪の悩み（複数選択可）
- サロン選択の理由
- 過去の施術履歴（黒染め・ブリーチ・縮毛矯正等）
- 店内での過ごし方の希望
- 嫌いな接客スタイル

### 5-2. 2回目以降の来店（チェックイン）

1. 店舗のQRコードをLINEで読み取る
2. LINE認証 → チェックイン完了画面が表示
3. 「Check IN ありがとうございます」メッセージと「スタッフがお声がけするまで少々お待ちください」案内
4. 「閉じる」ボタンでLINE画面に戻る

### 5-3. コンセプトアンケート

コンセプトメニュー（管理者がメニュー設定で`is_concept`をONにしたメニュー）の施術前に、顧客に追加アンケートを実施します。カウンセリングフォームとして機能します。

### 5-4. アンケート・ルーレット

来店後にLINEで送信されるアンケートリンクから回答。回答後にルーレット画面が表示され、当選確率（管理者設定）に基づいて景品が当たります。

## 6. カルテ閲覧（iPad + QR認証）

スタッフが施術中に顧客カルテを閲覧するための機能です。

**アクセスURL**: https://kmt.cookie.hair/karte-qr

> ログイン不要。iPadのブラウザで直接アクセスできます。

### 利用手順

#### Step 1: QR表示（iPad側）
1. iPadで `/karte-qr` にアクセス
2. QRコードが画面に表示される
3. 「認証待ち…」のスピナーが表示される

#### Step 2: LINE認証（スタッフのスマホ側）
1. スタッフが自分のスマートフォンのLINEでQRを読み取る
2. LINE認証が自動実行される
3. スタッフアカウントの確認（staffテーブルに登録されているか）
4. GPS位置確認（店舗の許容半径内か）
5. 「認証完了」画面が表示される → スマホ側は自動で閉じる

#### Step 3: カルテ閲覧（iPad側が自動切替）
1. iPad側が自動的に顧客検索画面に切り替わる
2. 名前・フリガナ・顧客コードで顧客を検索
3. 顧客をタップすると、カルテ詳細が表示される

**閲覧できる情報**: 基本情報・来店履歴・カルテ（アンケート回答）・ケース記録・LINE送信履歴

> **注意**: 閲覧専用です。編集・削除はできません。

**QRの有効期限**: 10分間。期限切れの場合は「QRを更新する」ボタンで再生成できます。

**GPS範囲外の場合**: 「店舗の範囲外です（○○m）。店舗内でのみ使用できます。」と表示され、認証できません。

## 7. LINE自動配信

以下のLINEメッセージが自動配信されます。すべてのテンプレートは管理者が設定画面から編集できます。

| 種類 | タイミング | 内容 |
|------|-----------|------|
| サンキューLINE | 施術ログ保存時に即時送信 | 来店のお礼メッセージ |
| リマインド① | スタイル設定の「リマインダー①日数」後 | 再来店のご案内 |
| リマインド② | スタイル設定の「リマインダー②日数」後 | 再来店のご案内（クーポン付き） |
| 休眠顧客LINE | 休眠しきい値（デフォルト90日）超過時 | 再来店促進メッセージ |
| メンテナンス① | 設定日数後 | メンテナンスチケット通知 |
| メンテナンス② | 設定日数後 | メンテナンスチケット通知（2回目） |

**送信条件**: LINE連携済み かつ ブロックされていない顧客にのみ送信されます。

**送信結果**: ダッシュボードの「LINE配信状況テーブル」、および顧客詳細の「LINE送信履歴」タブで確認できます。

## 9. 補足・運用Tips

### デモモード

Supabase（データベース）が未接続の環境では、すべてのページがデモデータで動作します。実データなしでUIの確認が可能です。

### 顧客のLINEブロック

顧客がLINE公式アカウントをブロックすると、自動的にシステム上で「ブロック」状態になり、以降のLINE配信がスキップされます。ブロック解除時に自動復帰します。

### GPS設定のポイント

- 店舗設定で緯度・経度・許容半径を正しく設定してください
- 半径は建物のサイズ + 若干の余裕を持たせてください（推奨: 100〜200m）
- GPS機能自体をOFFにすることもできます（タイムカード設定画面）

### AI要約機能

ケース記録にAI要約機能があります。施術の悩み・施術内容を入力すると、AIが要約を自動生成します。Anthropic Claude または OpenAI GPT が使われます。

### パスワードについて

- **ログインパスワード**: 管理者が初回登録時に仮パスワードを設定 → スタッフが初回ログイン時に変更
- **編集・削除確認パスワード**: ログインパスワードと同一
- **雇用形態ページパスワード**: 別途設定（システム設定で変更可能）
